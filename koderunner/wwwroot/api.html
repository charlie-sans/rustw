WebSocket Endpoints

WebSocket

/code

Handles code submission and processing.

Expected Message Format for something like C:

/*
File_name: main.c
Project: MyProject
Build: gcc -o main main.c
Run: ./main 
*/
// Your code here

// noting that build and run are optional and can be omitted
WebSocket

/PMS

Project Management System endpoint.

Expected Message Format:

{
    "PMS_System": "1.2.0",
    "Project_Name": "MyProject",
    "Main_File": "main.c",
    "Project_Build_Systems": "cmake",
    "Project_Output": "main",
    "Run_On_Build": "True"
}
WebSocket

/stop

Controls process termination.

Expected Message Format:

{
    "stopped": true
}
WebSocket

/terminput

Handles terminal input commands.

Usage:

Send terminal commands as text messages through this WebSocket connection.
