KodeRunner Documentation

Welcome to the KodeRunner documentation. This document provides a high-level overview of the classes and files defined in the codebase.

Core.cs

Defines the core settings and configuration paths for the KodeRunner project.

  • GetVersion(): Returns the version of the assembly.
  • RootDir: Base directory for all KodeRunner files.
  • ConfigPath: Path to the configuration file.

TerminalCodeParser.cs

Parses terminal output and converts ANSI codes to HTML color codes.

  • ParseToResonite(): Parses ANSI codes in the input string.
  • ParseColorCode(): Converts ANSI color codes to HTML color tags.

IRunnable.cs

Defines the interface and attributes for runnable classes.

  • IRunnable: Interface for all runnables.
  • RunnableAttribute: Attribute to be used on classes that implement IRunnable.

Provider.cs

Manages settings and configurations for runnables.

  • ISettingsProvider: Interface for the settings provider.
  • SettingsProvider: Struct implementing the ISettingsProvider interface.

BuildProcess.cs

Handles the build process for various programming languages.

  • BuildProcess: Class that executes the build process.
  • SetupCodeDir(): Sets up the necessary directories for the build process.
  • SendInput(): Sends input to the build process.

TerminalProcess.cs

Manages terminal input and output.

  • ExecuteCommand(): Executes a command in the terminal.
  • SendInput(): Sends input to the terminal process.
  • StopAllProcesses(): Stops all active terminal processes.

RunnableManager.cs

Loads and executes runnable code snippets.

  • LoadRunnables(): Loads runnables from the current domain assemblies.
  • ExecuteAll(): Executes all registered runnables.
  • ExecuteFirstMatchingLanguage(): Executes the first runnable matching the specified language.

Program.cs

Main entry point for the KodeRunner application.

  • Main(): Starts the HTTP and WebSocket server.
  • HandleCodeWebSocket(): Handles WebSocket connections for code submissions.
  • HandlePmsWebSocket(): Handles WebSocket connections for project management system.
  • HandleStopWebSocket(): Handles WebSocket connections for stopping processes.

Envars.cs

Manages environment variables for the KodeRunner project.

  • Envars: Class that reads environment variables from a .env file.
